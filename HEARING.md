# お客さまへのヒアリング事項

**最終更新**: 2026年5月26日

開発を進める中で「お客さまのご意向を確認させていただきたい」点をまとめております。
ご打ち合わせのうえ方針を決めていただきましたら、本ファイルから該当項目を「決定済み」セクションに移動いたします。

---

## 1. 受注の状態管理

**現状**: 受注の状態を以下の4つで管理

> 見積 / 受注確定 / 出荷済 / キャンセル

- [ ] このセットで十分か、もっと細かく分けたいか
  - 例: 「受注確定」と「出荷準備中」と「出荷済」を区別する など
- [ ] 新規登録時に最初にセットされる状態は「見積」で良いか
- [ ] 製造管理画面に出すのは「受注確定」以降の案件だけにする想定だが、見積段階から見たい場面があるか

---

## 2. 製造工程の流れ

**現状**: 以下の流れを標準として用意

> 受注 → 設計 → 板金 → 塗装 → 組み立て → 納品 / 遅延

- [ ] このフローで御社の実態に合っているか（順番・名称）
- [ ] 「遅延」の扱い
  - 現状: 工程の一つとして登録（一番後ろ）
  - 案: 「特別な状態」として通常の工程とは別扱いに変えるか
- [ ] 工程の追加・変更を操作できる人を、管理者だけに限定する必要があるか
- [ ] 製品の種類によって異なる工程フローが必要か
  - 例: A製品ラインは「受注 → 設計 → 組立 → 検査」、B製品ラインは「受注 → 板金 → 塗装 → 組立」など

---

## 3. 各工程のチェックリスト

**現状**: 各工程に「タスク（チェック項目）」を持たせられる仕組みあり

- [ ] 各工程で標準的にチェックすべき項目を事前に定義しておく必要があるか
- [ ] 作業者が自由にチェック項目を追加できる運用にするか、管理者だけが追加するか
- [ ] チェックの証跡として、以下も記録する必要があるか
  - 写真の添付
  - コメント・備考
  - （現状は「誰が」「いつ」のみ記録）

---

## 4. 工程ごとの日程

**現状**: 製造1件ごとに、工程ごとに開始日・納期を入力できる。
ガントチャート上でドラッグして直接日付を動かせる。

- [ ] 受注を登録した時に、各工程の予定日程を **自動で割り振る** 仕組みがあると便利か
  - 現状: 受注時は空欄、ガントチャート上で後から埋めていく
- [ ] 「予定の日程」と「実績の日程」を **両方記録して比較できる** ようにするか
  - 現状: 1つの日程のみ記録

---

## 5. ガントチャートの表示

**現状**: 表示期間（2週・1ヶ月・3ヶ月・6ヶ月・1年）と粒度（日・週・月）を切り替え可能

- [ ] バーに「この工程はチェックリストの何%完了したか」を表示するか
  - 例: チェックリスト5項目中3項目完了 → 60%
- [ ] ガントで工程をドラッグして日付を変えた時、後ろの工程も連動して動かすか
  - 現状: 動かした工程のみ変更

---

## 6. 不足・不良の履歴

**現状**: 入庫時に「○個足りない」「△個不良」を記録すると、履歴として残る

- [ ] 履歴データはいつまで残しておくか
  - 無期限 / 1年で削除 / 解決後30日で削除 など
- [ ] 月次・年次のレポートとして集計したいか
  - 例: 「今月の欠品件数」「仕入先別の不良率」など
- [ ] 仕入先ごとの **不良率・遅延率を集計する画面** が必要か

---

## 7. メーカー欠品からの振替発注（新機能）

**現状**: 仕入先が欠品で出せない時、別の仕入先へ振替発注を作成できる。
振替元と振替先の発注は双方向にリンクされる。

- [ ] 振替先の仕入先を選ぶ時、過去の発注実績や単価を参考表示すると便利か
  - 現状: 仕入先マスタからの選択のみ
- [ ] 振替後の「元の発注」をどう扱うか
  - A. 欠品マークだけ付けて残す（現状）
  - B. 自動的にキャンセルする
  - C. そのまま「納品待ち」として追いかけ、後から仕入先が回復したら受け取る
- [ ] 振替が頻発する場合の運用ルール（誰が判断するか、上長承認が必要か等）

---

## 8. 製品管理表（新メニュー）

**現状**: 桐の「営業 製品管理表」と同じ構成で、入力欄のみ用意（保存機能未実装）

- [ ] 項目の **追加・削除・並び替え** のご希望はあるか
- [ ] データはどう保存するか
  - 案A: 既存の受注・製造データと自動連携（入力不要な項目は表示のみ）
  - 案B: このフォーム独立で入力・保存できるようにする
- [ ] **自動計算** の対象範囲
  - 例: 製造費計（設計費+板金費+...）／総合計／工数の自動積算 など
- [ ] 一覧画面・検索画面の必要性
- [ ] 印刷・PDF出力の必要性

---

## 9. 複数拠点・複数事業所への対応

- [ ] 本システムを **複数の顧客企業に提供** する想定はあるか
- [ ] 工程マスタ・受注ステータス等を **企業ごと・拠点ごとに分けて** 管理する必要があるか
- [ ] ユーザーと所属企業・所属事業所の紐付けが必要か

---

## 10. ユーザーごとの権限

**現状**: ログインユーザーであればほぼ全機能を操作可能（管理者・一般の区別が緩い）

- [ ] 以下の操作は、どの役職・部門の方まで許可するか
  - 工程マスタの編集
  - 受注・製造のステータス変更
  - 発注の承認
  - 振替発注の作成
  - 不足/不良履歴の編集・削除
  - 部品マスタ・仕入先マスタの編集

---

## 11. 通知

**現状**: 通知機能なし

- [ ] 以下のイベント発生時に通知を出したいか
  - 納期遅延が発生／納期が近づく
  - 在庫切れ・発注点割れ
  - 不足・不良の発生
  - 振替発注が作成された
  - 製造完了
- [ ] 通知の手段（メール / Slack / 画面内表示 / スマホ通知 等）
- [ ] 通知先（担当者・上長・部門全員 など）の指定ルール

---

## 12. その他のご相談点

- [ ] 印刷・帳票出力の必要性（発注書・納品書・現品票・棚卸表 等）
- [ ] バーコード・QRコードの運用範囲（入庫だけか、出庫・棚卸も含むか）
- [ ] バックアップ・データ移行の方針
- [ ] 既存システム（桐 等）との連携・移行の必要性
- [ ] 端末（PC / タブレット / スマホ）ごとの利用想定
