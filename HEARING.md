# クライアントへのヒアリング事項

開発を進める中で「これは要確認」と判断した項目をまとめます。次回ヒアリング時に確認し、決定したら本ファイルから削除（または「決定済み」セクションに移動）してください。

## 1. 受注ステータス（salesStatus）

現状実装: `quote(見積) / confirmed(受注確定) / shipped(出荷済) / cancelled(キャンセル)` の4種類。

- [ ] このセットで十分か？
- [ ] 「受注確定」「出荷準備中」「出荷済」のような中間ステータスは必要か？
- [ ] 製造管理画面に表示する条件: 現状は「受注確定」以降のみとする想定だが、見積段階から見たい運用があるか？
- [ ] 新規登録時のデフォルトステータス: 現状 `quote(見積)` だが、ヒアリングのうえ確認

## 2. 製造工程（MProductionStage）

現状実装: デフォルトで以下7工程を seed
受注 → 設計 → 板金 → 塗装 → 組み立て → 納品 / 遅延

- [ ] このフローでクライアント企業の実態に合うか？
- [ ] 「遅延」をステータスとして扱うか、別フラグとして扱うか（現状は1工程として登録、sortOrder=99）
- [ ] 工程の追加・並び替え・色変更は管理者のみ操作可能とする権限制御が必要か？
- [ ] 複数の製造ライン（例: A製品ラインとB製品ライン）で異なる工程フローが必要になるか？
  - 必要なら: 製品マスタやカテゴリと工程マスタを紐付ける拡張が必要

## 3. 工程ごとの作業項目（MProductionTask / チェックリスト）

- [ ] 各工程のタスク（チェックリスト）の標準セットは事前定義しておく必要があるか？
- [ ] 作業者が任意でチェック項目を追加できる運用か、管理者のみが追加するのか？
- [ ] チェック完了の証跡（写真・コメント・実施者・実施日時）の記録は必要か？
  - 現状はチェック状態・実施者・実施日時のみ記録。コメントや写真は未対応

## 4. 工程ごとの日程（TProdOrderStage）

- [ ] 受注登録時に各工程の予定日程を自動配分する仕組みが必要か？
  - 現状: 受注時は空、ガント上でドラッグして埋める運用
- [ ] 「予定 vs 実績」の二重バー表示が必要か？（提示された参考画面のような淡色＝予定 / 濃色＝実績）
  - 現状: 1本のバーのみ

## 5. ガントチャート

- [ ] 表示期間（日/週/月）の切替が必要か？
  - 現状: 2週/1ヶ月/3ヶ月/6ヶ月/1年で切替
- [ ] バーに進捗率（タスクチェック比率）を表示するか？
  - 現状: 表示なし。実装可能（各工程のタスクチェック数 / タスク総数）
- [ ] ガント上のドラッグで日程変更したとき、関連工程（後続）も連動して動かすか？
  - 現状: 動かした工程のみ変更

## 6. 不足/不良履歴（TOrderDetailShortage）

- [ ] 履歴データの保持期間は？（無期限 / 1年で削除 / 解決後30日で削除 など）
- [ ] 月次/年次レポートで集計可能にする必要があるか？
- [ ] 仕入先別の不良率・遅延率を集計する画面が必要か？

## 7. メーカー欠品からの振替発注

- [ ] 振替先の仕入先を選ぶ際、過去実績や単価情報を参考にしたい
  - 現状: 仕入先マスタから選択のみ
- [ ] 振替発注時に「元発注をどう扱うか」（欠品マークのみ / 自動キャンセル / そのまま追っかけ）の運用ルール

## 8. マルチテナント / 企業別カスタマイズ

- [ ] このシステムを複数の顧客企業に提供する想定か？
- [ ] 工程マスタ・受注ステータス等を企業ごとに分離する必要があるか？
- [ ] ユーザー（MUser）と企業の紐付けが必要か？

## 9. 権限（admin / manager / operator / viewer）

- [ ] 工程マスタ編集はどのロールまで可能か？
- [ ] 受注/製造のステータス変更はどのロールまで可能か？
- [ ] 詳細画面の編集はどのロールまで可能か？
- [ ] 現状はロールチェックがほぼ無く、ログインユーザーなら全操作可能

## 10. 通知

- [ ] 納期遅延・在庫切れ・不足発生時の通知（メール/Slack/Push）が必要か？
- [ ] 通知先（担当者割り当てルール）の設計が必要

### 補足

- 現状はデモ向け実装のため、上記が決まり次第、本実装に組み込みます
- 「最低限こうあって欲しい」「将来的にはこうしたい」を区別してヒアリングするとスムーズです
